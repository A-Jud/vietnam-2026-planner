nitty app for my trip that you can use too

מתכנן הטיולים החכם (Smart Trip Planner)

מתכנן טיולים גנרי, חכם ומבוסס AI המאפשר למשתמשים לבנות, לערוך ולנהל מסלולי טיול מורכבים לכל יעד בעולם. המערכת משלבת ממשק משתמש אינטואיטיבי, מפה אינטראקטיבית (Leaflet), חיבור מובנה למפות גוגל (Google Maps), ומנועי בינה מלאכותית (OpenAI או Gemini) ליצירת המלצות מותאמות אישית.

תכונות מרכזיות

בניית מסלול אוטומטית מבוססת AI: יצירת מסלול שלם ממספר ימים מוגדר, כולל חלוקה לוגית של יעדים, אטרקציות ואפשרויות לינה.

עריכה ידנית וידידותית: הוספה, עריכה ומחיקה של ימים, כולל ניהול יעדים מרכזיים, תיאור יומי, ופרטי לינה (הכל בחלונית עריכה אחת מרוכזת).

שדרוג יומי חכם (AI Magic):

✨ טיפים: קבלת המלצות למקומות סודיים, פנינים נסתרות ונקודות תצפית (ללא הצעות אוכל).

🍜 אוכל מקומי: המלצות למנות מקומיות ומסעדות ספציפיות פעילות, כולל קישורים לתפריטים.

🏨 מלונות: קבלת אפשרויות לינה מגוונות (תקציב, יוקרה, משפחות) ושיבוץ מהיר ליום הטיול.

🎯 אופציות למסלול: יצירת אפשרויות פעילות נוספות לבחירה וסימון הפעילויות הנבחרות.

🤖 תכנון מחדש: מחיקת הלו"ז הקיים וג'ינרוט מסלול חדש לחלוטין לאותו היום על בסיס בקשה ספציפית ("vibe").

ניווט חכם (Smart Routing): בלחיצת כפתור אחת, המערכת אוספת את היעדים המרכזיים, הפעילויות שנבחרו והטיפים השמורים, ומייצרת מסלול הליכה/נסיעה מרובה נקודות במפות גוגל. הניווט יכול להיפתח בתוך חלונית באפליקציה או בדפדפן חיצוני.

מפה אינטראקטיבית מובנית: תצוגה גיאוגרפית של מסלול הטיול (עם פוליגונים המחברים את היעדים) ושל נקודות העניין הספציפיות בכל יום (סמנים כתומים).

ניהול מפתחות API ופרומפטים (Admin Panel):

תמיכה במפתחות API של OpenAI (כולל מודל GPT-5.2 ו-gpt-4o-mini) ו-Google Gemini.

שמירת מפתחות API באופן מאובטח (Web Crypto API) בזיכרון המקומי (localStorage).

שליטה ועריכה של כל הפרומפטים (הוראות ה-AI) המשמשים את המערכת.

גיבוי ענן וייצוא מקומי:

חיבור לחשבון Google לשמירה וטעינה אוטומטית של המסלול מ-Google Drive.

ייצוא (הורדה) של קובץ JSON למחשב וייבוא שלו חזרה בעת הצורך.

מבנה המערכת

האפליקציה בנויה כולה כקובץ HTML יחיד (index.html) המכיל את כל קוד ה-HTML, ה-CSS וה-JavaScript הדרושים להפעלתה.

עיצוב: Tailwind CSS מובנה באמצעות תגית <script>.

מפות: ספריית Leaflet.js המשולבת עם OpenStreetMap (Nominatim) להמרת כתובות (Geocoding) לקואורדינטות.

אבטחה: שימוש ב-Web Crypto API (AES-GCM) להצפנת מפתחות ה-API בזיכרון המקומי.

תצוגה מקדימה למפות: שימוש ב-iFrame המעבד בקשות חיפוש וניווט והופך אותן לפורמט הנתמך על ידי Google Maps Embed API.

דרישות קדם והפעלה

מכיוון שהאפליקציה רצה כולה בצד הלקוח (Client-Side), אין צורך בשרת מיוחד או התקנות מורכבות:

שמרו את הקובץ כ-index.html.

פתחו את הקובץ בכל דפדפן מודרני (Chrome, Firefox, Safari, Edge).

על מנת שתכונות ה-AI יעבדו, יש צורך במפתח API תקין של OpenAI או Google Gemini.

מדריך למשתמש

הגדרת מפתח AI:

לחצו על כפתור גלגל השיניים (הגדרות) בפינה השמאלית העליונה.

עברו ללשונית "OpenAI" או "Gemini" בהתאם למפתח שברשותכם.

הדביקו את המפתח, בחרו את המודל הרצוי וודאו שהספק הפעיל מסומן נכון למטה.

לחצו על "שמור הגדרות".

יצירת מסלול מבוסס AI:

לחצו על הכפתור "🤖 בניית מסלול AI".

הזינו את יעד ההתחלה (למשל: "פריז"), מספר הימים ודגשים מיוחדים (אופציונלי).

המתינו עד לסיום התהליך ובחרו אם להחליף את המסלול הקיים או להוסיף את הימים לסוף המסלול.

עריכה ידנית:

לחיצה על סמל העיפרון ליד שם היום תפתח חלון עריכה שבו ניתן לשנות את שם היעד, תיאור, מלון ויעדים מרכזיים.

לחיצה על הכיתוב "🏨" בכרטיסיית היום תפתח ישירות את חלון העריכה במיקוד על אזור הלינה.

שימוש בכלי ה-AI היומיים:

השתמשו בכפתורים בתחתית כל יום (✨ טיפים, 🍜 אוכל, 🏨 מלונות, 🎯 אופציות) כדי להעשיר את המסלול בתוכן ספציפי ליעד.

ניתן למחוק או להעביר פריטים אלו לימים אחרים.

ניווט:

לחיצה על כפתור "🗺️ נווט" תייצר מסלול. בחרו אם להציג אותו במסך המפוצל או לפתוח אותו בדפדפן (מומלץ במובייל).

שמירה וגיבוי:

מומלץ ללחוץ על "📥 ייצוא" בסוף העבודה כדי לשמור גיבוי של המסלול במחשב שלכם.

הערות למפתחים (Developers Notes)

Geocoding Limit: פונקציית ה-Geocoding משתמשת בשירות החינמי של Nominatim (OSM). הוגדר תור (geocodeQueue) עם השהיה (Timeout) כדי להימנע מ-Rate Limiting של השרת.

Fallback API Key: במידה וברצונכם להגדיר מפתח גיבוי שיהיה זמין לכל המשתמשים כברירת מחדל, השתמשו בכלי ההצפנה שבהגדרות והחליפו את הערך של המשתנה FALLBACK_OAI_KEY בקוד המקור. חובה לוודא שהמשתנה activeProvider מוגדר לספק התואם למפתח זה (ראו פונקציה getDecryptedConfig).

Google Drive API: על מנת לאפשר גיבוי בענן, יש להחליף את CLIENT_ID במפתח אמיתי שנוצר ב-Google Cloud Console עם ההרשאה https://www.googleapis.com/auth/drive.file.
